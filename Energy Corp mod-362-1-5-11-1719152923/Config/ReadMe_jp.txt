このドキュメントでは、Configフォルダー内のファイルとフォルダーについて説明します。
特定のファイルを検索するには、ショートカットCtrl+Fを使用します。

0.LCIdと比重システム
LcIdは「Mod ID + Abnormality ID」を「,」で区切って構成されます。
各LcIdは改行 (Enter) で区切られます。
MODのインストール後、ゲームを実行すると、ConfigフォルダーにallCreatureList.txtという名前のファイルが生成されます。
このファイルには、MODのアブノーマリティーを含むほぼすべてのアブノーマリティーのLcIdが含まれています。このファイルからアブノーマリティーのLcIdを見つけてコピーするだけです。

Weightノード内には、「LcId + Weight」が存在する必要があります。
「Weight」は0～1の浮動小数点値で、値を減らすと出現確率が低くなり、増やすと出現確率が高くなります。
アブノーマリティがランダムに選択されるとき、それがWeightノードに含まれている場合、その重みは宝くじを引く確率に変換されます。
選択すると、アブノーマリティが表示されます。それ以外の場合は、同じリスクレベルリストから別のランダムなアブノーマリティが1回抽出されます。
そのため、アブノーマリティのWeightが0であっても、わずかながら出現する可能性があります。

1.applymod.xml
MODを適用または無効にします。

	1) 追加収容(addIsolate)
	このMODは収容室を追加するMODです。
	レガシーバージョンの追加の収容室を実装しました。
	ツール型アブノーマリティーは追加抽出されません。
	データ型: ブール値、デフォルト: true

	2) tt05(tt05)
	このMODはTT0.5MODです。ゲームスピードの値を変更できます。
	データ型: ブール値、デフォルト: true

	3) エネルギー精製(energyRefine)
	このMODはエネルギー精製MODです。レガシー版のエネルギー精製作業を実装しました。
	データ型: ブール値、デフォルト: true

	4) ランダム再配置化(isolateRandomize)
	このMODは収容室の再配置MODです。
	7日目以降は、一日の終わりにアブノーマリティーの位置がランダムに並び替えられます。
	データ型: ブール値、デフォルト: false

	5) 過負荷強化(overloadPlus)
	このMODはクリフォト過負荷強化MODです。
	ツール型アブノーマリティーを除くすべてのアブノーマリティーは、以前よりも高いクリフォト過負荷が付与されます。
	データ型: ブール値、デフォルト: true

	6) 異状(aberration)
	このMODはアブノーマリティーの亜種を追加するMODです。
	データ型: ブール値、デフォルト: true

	7) 中央階段(centralStair)
	このMODは、中央本部チームの階段追加MODです。透明な階段を通って中央本部チームをすぐに横切ることができます。
	データ型: ブール値、デフォルト: false

	8) 言語互換性(languageCompatibility)
	このMODは言語互換性MODで、これを使用すると、まだ翻訳されていない言語設定でEnergy CorpのMODを実行できるようになります。
	データ型: ブール値、デフォルト: true

2.modconfig.xml
MODの詳細を変更できます。

	1) 追加収容（addIsolate）
	これは収容室の追加MODです。管理する必要があるアブノーマリティーの数が増加します。

		ランダム抽出（randomExtraction）
		これはランダムにアブノーマリティーを追加するMODです。追加のアブノーマリティーは選択に関係なくランダムに抽出されます。
		データ型: ブール値、デフォルト: false

	2) tt05（tt05）
	TT0.5modです。ゲームスピードの値を変更できます。
	上から一時停止、通常速度、1.5倍速、2倍速の順です。

		(1)一時停止（pause）
		データ型：Float、範囲：0以上、デフォルト：0.5
		
		(2)スピードLv1（speedLv1）
		データ型：Float、範囲：0以上、デフォルト：1
		
		(3)スピードLv2（speedLv2）
		データ型：Float、範囲：0以上、デフォルト：1.5
		
		(4)スピードLv3（speedLv3）
		データ型：Float、範囲：0以上、デフォルト：2

	3) エネルギー精製（energyRefine）
	これはエネルギー精製作業MODです。

		(1) マックスLOBペナルティ（maxLobPenalty）
		LOBポイントペナルティの最大値。
		データ型：整数、範囲：0以上、デフォルト：8

		(2) 精製効率（refineEfficiency）
		エネルギー精製作業速度。数値が大きいほど、エネルギー精製作業はより速く完了します。
		データ型: 浮動小数点数、範囲: 0 以上、デフォルト: 15

		(3)ボス戦を無視する（ignoreBossFight）
		コア抑制中にエネルギーが全て集まった場合は、コア抑制を無視してエネルギー精錬作業を進め、
		この場合、コア抑制は失敗します。
		データ型: ブール値、デフォルト: false

	4) 過負荷強化（overloadPlus）
	これはクリフォト過負荷強化MODです。作業が完成するたびに、
	成功率は、リスクレベルごとに設定した数値だけ減少します。

		(1)zayin
		データ型：整数、範囲：0以上、デフォルト：4
	  
		(2)teth
		データ型：整数、範囲：0以上、デフォルト：5
	  
		(3)he
		データ型：整数、範囲：0以上、デフォルト：6
	  
		(4)waw
		データ型：整数、範囲：0以上、デフォルト：7
	  
		(5)aleph
		データ型：整数、範囲：0以上、デフォルト：8

	5) 異状（aberration）
	これはアブノーマリティーの亜種を追加するMODです。

		(1) 選択可能（selectable）
		アブノーマリティー選択フェーズでアブノーマリティーが出現するかどうか。
		データ型: ブール値、デフォルト: true

		(2) 擬態のブラックリスト（transformBlacklist）
		O-05-47-1はリスト内のアブノーマリティに変身しません。
		データ型: LcId、デフォルト:
		100019
		100022
		100024
		Qquickness_EnergyCo、178801
		Qquickness_EnergyCo、178805

4.ランダム抽出（randomExtraction）
ランダムでアブノーマリティーが追加されるMOD専用の設定。

	1)blacklist.xml
	特定のMODのアブノーマリティがランダムに出現するのを防ぐことができます。

		(1) ブラックリストを使用する（useBlacklist）
		Modブラックリスト機能を使用するかどうか。
		データ型: ブール値、デフォルト: false

		(2) ブラックリスト（blacklist）
		ここにブロックしたいMODのModIDを書き留めることができます。
		複数のModをブロックしたい場合は、ModIDを区切って改行（Enter）してください。
		データ型: 文字列

	2) dailyDifficulty.xml
	ランダムでアブノーマリティーを追加するMODの難易度を調整します。
	Zayin、Teth、He、Waw、Alephの順。数値が大きいほど抽出される可能性が高くなります。

	3) reserveCreature.xml
	希望の日に、希望の確率で希望のアブノーマリティに出会うことができます。
	「D+希望日」ノードを作成します。
	ランダムでアブノーマリティーを追加しない日は出現しません。


	4) risklevel.xml
	アブノーマリティIDを追加または削除することで、ランダム抽出の確率を高めたり排除したりできます。
	危険度が低いノードのアブノーマリティーは、ゲーム開始直後に抽選されやすくなりますが、ゲームが進むにつれて抽選されにくくなります。
	危険度の高いノードのアブノーマリティーはゲーム開始時は抽選されませんが、ゲームが進むにつれて抽選されやすくなります。

		(1) Creating a risklevel.xmlファイル
		RiskLevelフォルダー内に、MODが認識できるように名前に「risklevel」を含む.xmlファイルを作成します。
		ファイルの構造については、既存のrisklevel.xmlを参照してください。これには、preventRandomExtractionノードとWeightノードが1つずつ含まれている必要があります。
