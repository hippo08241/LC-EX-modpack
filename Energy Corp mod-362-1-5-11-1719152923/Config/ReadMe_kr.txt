에너지주식회사 모드의 Config 폴더 내의 파일과 폴더에 대해 설명합니다.
파일 이름을 보시고 ctrl + F 키로 원하는 항목을 찾아보세요.

0. LcId와 비중 시스템
LcId는 '모드 아이디 + 환상체 아이디'로 이뤄져 있으며, xml 안에서 ','으로 구분됩니다.
LcId 끼리는 엔터로 구분합니다.
모드 설치 후 게임 실행 시 Config 폴더에 생성되는 allCreatureList.txt 에 모드 환상체를 포함한
거의 모든 환상체들의 LcId가 적혀있으니 찾아서 복붙하시면 됩니다.

weight 노드 안에는 'LcId + 비중'이 들어가야 합니다.
'비중'은 0 이상 1 이하의 Float 값으로, 낮아지면 등장 확률이 낮아지고, 높아지면 등장 확률이 높아집니다.
환상체가 무작위로 선택될 때, weight 노드에 포함된 환상체인 경우 비중을 백분률의 당첨 확률로 바꿔 제비뽑기를 실행합니다.
당첨된 경우 그 환상체가 등장하고, 그렇지 않다면 한 번만 같은 위험 등급의 환상체 목록에서 다시 무작위 환상체를 뽑습니다.
따라서 어떤 환상체의 비중이 0이더라도 그 환상체가 등장할 확률이 조금 있습니다.

1. applymod.xml
모드를 적용하거나 해제합니다.

	1) addIsolate
	격리실 추가 모드입니다.
	레거시에 있었던 추가 격리실을 모드로 구현했습니다.
	도구형 환상체는 추가로 등장하지 않습니다.
	자료형: Boolean, 기본값: true

	2) tt05
	TT0.5 모드입니다. 게임의 배속 속도를 바꿀 수 있습니다.
	자료형: Boolean, 기본값: true

	3) energyRefine
	정제작업 모드입니다. 과거 레거시에 있던 정제작업을 모드로 구현했습니다.
	자료형: Boolean, 기본값: true

	4) isolateRandomize
	환상체 재배치 모드입니다.
	7일부터, 하루를 끝낼 때마다 환상체들의 위치가 무작위로 재배치됩니다.
	자료형: Boolean, 기본값: false

	5) overloadPlus
	클리포트 과부하 강화 모드입니다.
	도구형 환상체를 제외한 모든 환상체가 기존보다 수치가 높은 클리포트 과부하에 걸립니다.
	자료형: Boolean, 기본값: true

	6) aberration
	환상체 아종 추가모드입니다.
	자료형: Boolean, 기본값: true

	7) centralStair
	중앙본부 계단 모드입니다. 투명한 계단을 통해 중앙본부를 빠르게 횡단하실 수 있습니다.
	자료형: Boolean, 기본값: false

	8) languageCompatibility
	언어 호환 모드입니다. 에너지주식회사 모드가 번역되지 않은 언어 설정에서도 실행되게 해줍니다.
	자료형: Boolean, 기본값: true


2. modconfig.xml
여기서 모드의 세부사항을 변경할 수 있습니다.

	1) addIsolate
	격리실 추가 모드입니다. 관리해야 할 환상체의 수가 늘어납니다.

		randomExtraction
		무작위 추가 환상체 모드입니다. 추가 환상체가 당신의 선택 없이 무작위로 추출됩니다.
		자료형: Boolean, 기본값: false

	2) tt05
	TT0.5 모드입니다. 배속 모드에 따른 게임 속도를 조절할 수 있습니다.
	순서는 위에서부터 일시정지, 평상시, 1.5배속, 2배속입니다.

		(1) pause
		자료형: Float, 범위: 0 이상, 기본값: 0.5
		
		(2) speedLv1
		이 기능은 게임 내에서 속도를 바꿔줘야 작동하므로, 속도를 바꿀 수 없는 상황에서 작동하지 않을 수 있습니다.
		자료형: Float, 범위: 0 이상, 기본값: 1
		
		(3) speedLv2
		자료형: Float, 범위: 0 이상, 기본값: 1.5
		
		(4) speedLv3
		자료형: Float, 범위: 0 이상, 기본값: 2

	3) energyRefine
	정제작업 모드입니다.

		(1) maxLobPenalty
		LOB 포인트 패널티의 최대치입니다.
		자료형: Integer, 범위: 0 이상, 기본값: 8

		(2) refineEfficiency
		정제작업 속도입니다. 수치가 높을수록 정제작업이 빨리 끝납니다.
		자료형: Float, 범위: 0 초과, 기본값: 15

		(3) ignoreBossFight
		코어 억제 중 에너지가 다 모였다면, 코어 억제를 무시하고 정제작업을 진행합니다. 이 경우 코어 억제 실패입니다.
		자료형: Boolean, 기본값: false

	4) overloadPlus
	클리포트 과부하 강화모드입니다. 작업을 완료할 때마다, 위험 등급마다
	설정한 수치만큼 작업 성공률이 줄어듭니다.

		(1) zayin
		자료형: Integer, 범위: 0 이상, 기본값: 4
	  
		(2) teth
		자료형: Integer, 범위: 0 이상, 기본값: 5
	  
		(3) he
		자료형: Integer, 범위: 0 이상, 기본값: 6
	  
		(4) waw
		자료형: Integer, 범위: 0 이상, 기본값: 7
	  
		(5) aleph
		자료형: Integer, 범위: 0 이상, 기본값: 8

	5) aberration
	환상체 아종 모드입니다.

		(1) selectable
		아종이 환상체 선택지에 등장할지 여부입니다.
		자료형: Boolean, 기본값: true

		(2) transformBlacklist
		O-05-47-1이 목록의 환상체로 변신하지 않습니다.
		자료형: LcId, 기본값:
		100019

		Qquickness_EnergyCo, 178801
		Qquickness_EnergyCo, 178805

4. randomExtraction
무작위 추가 환상체 모드 전용 설정들입니다.

	1) blacklist.xml
	특정 모드의 환상체들이 무작위로 출현하는 걸 막을 수 있습니다.

			(1) useBlacklist
			모드 블랙리스트 기능의 사용 여부입니다.
			자료형: Boolean, 기본값: false

			(2) blacklist
			여기에 차단하고 싶은 모드의 모드 아이디를 적으시면 됩니다.
			여러개의 모드를 차단하고 싶은 경우, 모드 아이디를 엔터로 구분해주세요.
			자료형: String

	2) dailyDifficulty.xml
	무작위 추가 환상체 모드의 난이도를 조절합니다.
	zayin, teth, he, waw, aleph 순서입니다. 숫자가 커질 수록 나올 확률이 늘어납니다.

	3) reserveCreature.xml
	원하는 날짜에 원하는 확률로 원하는 환상체를 만나실 수 있습니다.
	'D + 원하는 날짜' 노드를 작성하시면 됩니다.
	무작위 환상체가 없는 날은 등장하지 않습니다.

	4) risklevel.xml
